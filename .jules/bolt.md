## 2025-02-28 - [String Parsing & Map Iteration Optimizations]
**Learning:** For optimizing hot paths like string parsing in `core/keyParser.js` (`parseKeySequence`), avoid intermediate array allocations (`.split('')`, spread syntax, `.map()`). Instead, iterate directly over characters via index tracking and append sequentially using inline loops to prevent garbage collection overhead. Additionally, when iterating over a Map's contents where only the values are required, prefer using `Map.prototype.values()` over `Map.prototype.entries()` to eliminate the performance overhead of allocating intermediate entry arrays and performing array destructuring.
**Action:** Always favor inline single-pass loops over functional array methods (`split`, `map`, spread) for hot string manipulation. When iterating Maps, explicitly use `.values()` or `.keys()` rather than `.entries()` if the complementary pair element is unused.
